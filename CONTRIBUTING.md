# Contributing to the LLM Go Local Sample

Contributions are welcome to this repository:
- Reporting Issues
- Suggesting Improvements
- Requesting Samples
- Requesting Clarifications

This contribution note contains the general guidelines for contributing.

## Code Of Conduct
We want to keep work and communication around this repository open, inclusive and respectful.

Please checkout the [github code of conduct](https://docs.github.com/en/site-policy/github-terms/github-community-code-of-conduct) to which this repository adheres to.

## Reporting issues
Issues related to the samples should be reported as github issues.

In the issue description, please make sure to details the issue as much as relevant.

Issue should be structured in the following format:
- **Given:** (Pre-conditions)
- **When:** (Actions)
- **Expected:** (What was expected to occur)
- **Actual:** (What actually occurred)

## Suggesting Improvements
Improvements should be suggested by creating an issue.

You can also submit directly a pull request of the improvement suggested.

## Requesting Samples
Sample requests should be made by creating an issue.

## Requesting Clarifications
Clarifications about samples can be requested in an issue.
